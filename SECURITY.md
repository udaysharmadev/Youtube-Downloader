# Security Policy

## Supported Versions

We recommend always running the latest version of the application from the `main` branch to ensure you have the latest security patches and dependency updates.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

Security is a top priority for us. If you discover a security vulnerability within YouTube Downloader, please do not disclose it publicly. 

Instead, please send a direct message to Uday Sharma via [Twitter/X](https://x.com/udaysharmatech) or [LinkedIn](https://www.linkedin.com/in/udaydotai/).

We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress. If you have not received a reply to your email within 48 hours, or have not heard from us for the past 5 days, please follow up with us.

### What to include in your report:
- Type of vulnerability (e.g., XSS, SQLi, RCE)
- Step-by-step instructions to reproduce the vulnerability
- Any potential impact on users or the application
- (Optional) Suggested mitigation or patch

Thank you for helping keep the open source community safe!
